Installation par lieu

Gateway caméras

Cette installation est à réaliser une fois dans chaque restaurant, boutique ou projet.

Matériel nécessaire

  1. Un mini PC Linux, Raspberry Pi 4/5 ou NVR compatible dans chaque lieu.
  2. Les adresses IP locales, marques, modèles et comptes administrateur des caméras.
  3. Une connexion internet stable. Ethernet est recommandé pour le gateway et les caméras.
  4. Tailscale installé sur le gateway, le téléphone, la tablette et l’ordinateur du propriétaire.

Compatibilité pour parler

  1. La caméra doit avoir un haut-parleur, pas uniquement un microphone.
  2. Rechercher ONVIF Profile T ou une compatibilité talk go2rtc pour le modèle exact.
  3. Le flux doit fonctionner en WebRTC avec `video+audio+microphone`.
  4. Le navigateur doit ouvrir le dashboard en HTTPS et autoriser le microphone.

1. Démarrer go2rtc avec Docker

services:
  go2rtc:
    image: alexxit/go2rtc:1.9.14
    network_mode: host
    restart: unless-stopped
    volumes:
      - ./go2rtc.yaml:/config/go2rtc.yaml:ro

Les fichiers exemples complets se trouvent dans le dossier camera-gateway/ du projet.

2. Ajouter les flux locaux

streams:
  cuisine-principale:
    - rtsp://UTILISATEUR:MOT_DE_PASSE@192.168.1.20:554/FLUX_CAMERA
  salle:
    - rtsp://UTILISATEUR:MOT_DE_PASSE@192.168.1.21:554/FLUX_CAMERA

api:
  listen: "127.0.0.1:1984"
  local_auth: true

Ce fichier reste uniquement dans le lieu. Il ne doit jamais être envoyé sur GitHub ou enregistré dans le dashboard.

3. Publier uniquement dans le réseau privé

sudo tailscale up
sudo tailscale serve --bg 1984
tailscale serve status

Tailscale affiche une adresse du type https://camera-bordeaux.nom-tailnet.ts.net. C’est cette adresse qui doit être ajoutée comme URL gateway.

4. Autoriser uniquement le propriétaire

{
  "groups": {
    "group:camera-owners": ["EMAIL_PROPRIETAIRE"]
  },
  "tagOwners": {
    "tag:camera-gateway": ["autogroup:admin"]
  },
  "grants": [{
    "src": ["group:camera-owners"],
    "dst": ["tag:camera-gateway"],
    "ip": ["tcp:443", "tcp:8555", "udp:8555"]
  }]
}
sudo tailscale up --advertise-tags=tag:camera-gateway

Ajoutez cette règle à la politique Tailscale existante, puis attribuez le tag à chaque boîtier. Les comptes cuisine et staff restent également bloqués par le dashboard.
